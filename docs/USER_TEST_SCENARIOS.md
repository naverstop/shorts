# 사용자 테스트 시나리오

**버전**: 1.0  
**일시**: 2026년 3월 3일  
**대상**: 쇼츠 영상 자동화 플랫폼 최종 사용자

---

## 📋 테스트 개요

이 문서는 실제 사용자(관리자 및 콘텐츠 크리에이터)가 시스템을 테스트할 수 있도록 구성된 시나리오입니다. 각 시나리오는 실제 업무 흐름을 따라 구성되었으며, 단계별로 수행하면서 시스템의 동작을 확인할 수 있습니다.

### 테스트 목표
- ✅ 시스템의 모든 핵심 기능이 정상 동작하는지 확인
- ✅ 사용자 경험 (UX)이 직관적이고 편리한지 평가
- ✅ 실제 YouTube/TikTok 플랫폼과의 연동 검증
- ✅ 오류 상황에서의 시스템 대응 확인

### 테스트 환경
- **Admin Dashboard**: http://localhost:3000
- **Backend API**: http://localhost:8001
- **Android Agent**: 실제 기기 또는 에뮬레이터

---

## 🎭 테스트 역할 정의

### 역할 1: 시스템 관리자 (Admin)
- 시스템 설정 및 모니터링 담당
- Agent 관리 및 Job 생성
- Platform Credential 설정
- Upload Quota 관리

### 역할 2: Android Agent 사용자
- Android 기기에서 앱 실행
- Job 자동 수신 및 영상 생성
- 완료 상태 확인

### 역할 3: 콘텐츠 크리에이터 (선택적)
- Dashboard에서 스크립트 생성
- 트렌드 분석 활용
- 영상 게시 관리

## 📱 시나리오 1: 첫 사용자 등록 및 시스템 탐색 (10분)

### 목표
신규 사용자가 시스템에 등록하고 기본 인터페이스를 둘러봅니다.

### 전제 조건
- 시스템이 실행 중 (START-UAT.bat 실행 완료)
- 브라우저 준비 (Chrome/Edge 권장)

### 단계별 진행

#### 1.1 회원가입
1. 브라우저에서 http://localhost:3000 접속
2. "회원가입" 버튼 클릭
3. 정보 입력:
   - 사용자명: `test_creator1`
   - 이메일: `creator1@test.com`
   - 비밀번호: `TestPass123!`
4. "회원가입" 버튼 클릭
5. **확인 사항**: 
   - ✅ "회원가입 성공" 메시지 표시
   - ✅ 자동으로 로그인 화면으로 이동

#### 1.2 로그인
1. 로그인 정보 입력:
   - 사용자명: `test_creator1`
   - 비밀번호: `TestPass123!`
2. "로그인" 버튼 클릭
3. **확인 사항**:
   - ✅ Dashboard 메인 화면으로 이동
   - ✅ 상단에 사용자명 표시
   - ✅ "로그아웃" 버튼 확인

#### 1.3 Dashboard 탐색
1. 화면 구성 확인:
   - **Header**: 사용자명, 로그아웃 버튼
   - **Health Status**: 시스템 상태 (API, Database, Redis)
   - **Agents**: Agent 목록 및 통계
   - **Jobs**: Job 목록 및 통계
   - **Platforms**: 지원 플랫폼 목록
   - **Credentials**: 인증 정보 관리
   - **Trends**: 트렌드 키워드
   - **Scripts**: AI 스크립트 생성
   - **Upload Quotas**: 업로드 할당량
2. 각 섹션을 클릭하여 데이터 확인
3. **확인 사항**:
   - ✅ 모든 섹션이 오류 없이 로드됨
   - ✅ 초기 상태에서는 대부분 데이터가 비어있음 (정상)

### 예상 소요 시간
10분

### 성공 기준
- 회원가입 및 로그인 완료
- Dashboard의 모든 섹션 확인
- 시스템 상태가 "healthy" 표시

### 문제 발생 시
- 회원가입 실패: 이메일 형식 확인, 중복 사용자명 확인
- 로그인 실패: 사용자명/비밀번호 정확히 입력 확인
- Dashboard 로드 안됨: Backend 서버 실행 상태 확인

## 🤖 시나리오 2: Android Agent 등록 및 Job 자동 처리 (30분)

### 목표
Android 기기를 Agent로 등록하고, 자동으로 Job을 처리하는 과정을 확인합니다.

### 전제 조건
- Android 기기 또는 에뮬레이터 준비
- Agent APK 설치 완료
- Dashboard 로그인 상태

### 단계별 진행

#### 2.1 Android Agent 앱 설정
1. Android 기기에서 "Shorts Agent" 앱 실행
2. 첫 화면에서 "Settings" 탭 이동
3. Backend URL 입력:
   - PC의 IP 주소 확인 (예: `192.168.1.100`)
   - `http://192.168.1.100:8001` 형식으로 입력
4. Device Name 입력: `My Test Phone`
5. "Register" 버튼 클릭
6. **확인 사항**:
   - ✅ "Registration successful" 메시지 표시
   - ✅ Agent ID 자동 생성 및 표시

#### 2.2 Dashboard에서 Agent 확인
1. Dashboard로 돌아가기
2. **Agents** 섹션 확인
3. **확인 사항**:
   - ✅ 방금 등록한 Agent가 목록에 표시됨
   - ✅ Device Name: "My Test Phone"
   - ✅ Status: "online" (녹색)
   - ✅ Last Heartbeat: 방금 전 시간

#### 2.3 Job 생성
1. Dashboard의 **Jobs** 섹션으로 이동
2. "Job 생성" 폼 작성:
   - **플랫폼**: YouTube 선택
   - **제목**: `테스트 영상 - 첫 번째`
   - **스크립트**: 
     ```
     안녕하세요! 이것은 테스트 영상입니다.
     쇼츠 자동화 시스템이 정상적으로 작동하는지 확인하고 있습니다.
     감사합니다!
     ```
   - **우선순위**: 5
3. "Job 생성" 버튼 클릭
4. **확인 사항**:
   - ✅ "Job 생성 성공" 메시지
   - ✅ Jobs 목록에 새 Job 표시
   - ✅ 상태: `pending` (노란색)

#### 2.4 Agent가 Job 자동 처리 (대기)
1. Android 앱의 "Home" 탭 확인
2. 약 30초 대기 (Agent가 자동으로 Job 폴링)
3. **확인 단계**:
   - **Step 1** (0-30초): App에 Job 수신 확인
     - "Assigned Jobs" 카운트 증가
     - Job 제목 표시
   - **Step 2** (30초-1분): 영상 렌더링
     - 상태: `rendering`
     - 진행률 표시 (있는 경우)
   - **Step 3** (1-2분): 파일 업로드
     - 상태: `uploading`
     - "Uploading video..." 메시지
   - **Step 4** (2-3분): 완료
     - 상태: `completed`
     - "Job completed successfully!" 메시지

#### 2.5 Dashboard에서 완료 확인
1. Dashboard의 Jobs 섹션 새로고침
2. 방금 생성한 Job 확인
3. **확인 사항**:
   - ✅ 상태: `completed` (녹색)
   - ✅ Job 클릭 시 상세 정보 확인
   - ✅ `video_path` 필드에 파일 경로 표시
   - ✅ `video_url` 필드에 다운로드 URL 표시

#### 2.6 생성된 영상 확인
1. Dashboard에서 `video_url` 링크 복사
2. 새 브라우저 탭에서 열기
3. **확인 사항**:
   - ✅ 영상 파일이 다운로드되거나 재생됨
   - ✅ 영상 길이 및 품질 확인

### 예상 소요 시간
30분 (대부분 Agent 처리 대기 시간)

### 성공 기준
- Android Agent 등록 완료
- Job 자동 수신 및 처리
- 영상 파일 생성 및 업로드 완료
- Dashboard에서 완료 상태 확인

### 문제 발생 시
- Agent 등록 실패: Backend URL 정확히 입력 확인, 네트워크 연결 확인
- Job 수신 안됨: Agent가 online 상태인지 확인, 30초마다 폴링하므로 대기 필요
- 영상 생성 실패: Agent 로그 확인, FFmpeg 설치 여부 확인 (fallback으로 placeholder 생성됨)

## 🎬 시나리오 3: YouTube 실제 게시 (20분)

### 목표
완료된 영상을 실제 YouTube 채널에 게시합니다.

### 전제 조건
- 시나리오 2 완료 (완료된 Job 존재)
- YouTube 계정 준비 (테스트용 채널)
- YouTube OAuth Credential 설정 완료

### 단계별 진행

#### 3.1 YouTube Credential 확인
1. Dashboard의 **Credentials** 섹션으로 이동
2. YouTube credential이 있는지 확인
3. 없다면 설정 필요 (사용 메뉴얼 참고)

#### 3.2 YouTube 게시
1. Dashboard의 **Jobs** 섹션
2. `completed` 상태인 Job 찾기
3. Job 우측의 **"YouTube 게시"** 버튼 클릭
4. 확인 창: "이 영상을 YouTube에 게시하시겠습니까?" → **확인**
5. 처리 대기 (10-30초)
6. **확인 사항**:
   - ✅ "YouTube 게시 완료" 메시지
   - ✅ Job 상세 정보에 `youtube_upload` 데이터 추가:
     - `video_id`: YouTube 영상 ID
     - `video_url`: YouTube 시청 URL
     - `published_at`: 게시 시각

#### 3.3 YouTube Studio 확인
1. 새 탭에서 https://studio.youtube.com 접속
2. "콘텐츠" 메뉴 클릭
3. **확인 사항**:
   - ✅ 방금 업로드된 영상이 목록에 표시됨
   - ✅ 제목: Job의 title과 동일
   - ✅ 상태: "업로드 완료" 또는 "처리 중"

#### 3.4 YouTube에서 영상 시청
1. Dashboard의 `video_url` 링크 클릭
2. YouTube 시청 페이지로 이동
3. **확인 사항**:
   - ✅ 영상이 정상 재생됨
   - ✅ 제목/설명 확인
   - ✅ 공개 상태 확인 (기본: private)

### 예상 소요 시간
20분

### 성공 기준
- YouTube에 영상 업로드 완료
- YouTube Studio에서 영상 확인
- 영상 URL로 시청 가능

### 문제 발생 시
- 게시 실패 (401): YouTube OAuth 재인증 필요
- 게시 실패 (403): YouTube Data API 할당량 초과 또는 권한 부족
- 게시 실패 (500): Backend 로그 확인

## 📊 시나리오 5: Upload Quota 관리 (15분)

### 목표
플랫폼별 업로드 할당량을 설정하고 관리합니다.

### 전제 조건
- Dashboard 로그인 상태

### 단계별 진행

#### 5.1 할당량 생성
1. Dashboard의 **Upload Quotas** 섹션
2. "할당량 생성" 폼 작성:
   - **플랫폼**: YouTube 선택
   - **일일**: `3`
   - **주간**: `15`
   - **월간**: `50`
3. "할당량 생성" 버튼 클릭
4. **확인 사항**:
   - ✅ "할당량 생성 완료" 메시지
   - ✅ 목록에 새 할당량 표시
   - ✅ 사용량: 0/0/0

#### 5.2 할당량 테스트 (업로드)
1. 시나리오 3을 3회 반복 (YouTube 게시 3회)
2. 각 게시 후 **Upload Quotas** 섹션 확인
3. **확인 사항**:
   - ✅ 1회 후: 사용 1/1/1
   - ✅ 2회 후: 사용 2/2/2
   - ✅ 3회 후: 사용 3/3/3

#### 5.3 할당량 초과 테스트
1. 4번째 YouTube 게시 시도
2. **확인 사항**:
   - ✅ 게시 실패
   - ✅ 에러 메시지: "Daily limit exceeded"

#### 5.4 수동 리셋
1. **Upload Quotas** 섹션
2. "Daily 초기화" 버튼 클릭
3. 확인 창: "정말로 daily 할당량을 초기화하시겠습니까?" → **확인**
4. **확인 사항**:
   - ✅ "daily 할당량 초기화 완료" 메시지
   - ✅ 사용량: 0/3/3

#### 5.5 리셋 후 재시도
1. 다시 YouTube 게시 시도
2. **확인 사항**:
   - ✅ 게시 성공
   - ✅ 사용량: 1/4/4

#### 5.6 할당량 삭제
1. 할당량 항목의 "삭제" 버튼 클릭
2. 확인 창: "정말로 이 할당량을 삭제하시겠습니까?" → **확인**
3. **확인 사항**:
   - ✅ "할당량 삭제 완료" 메시지
   - ✅ 목록에서 삭제됨

### 예상 소요 시간
15분 (YouTube 게시 시간 포함 시 더 길어질 수 있음)

### 성공 기준
- 할당량 생성/조회/수정/삭제 모두 정상 동작
- 할당량 초과 시 업로드 차단
- 리셋 후 다시 업로드 가능

### 문제 발생 시
- 할당량 초과인데 업로드됨: Backend 로그 확인, Quota 체크 로직 확인
- 리셋 안됨: API 응답 확인, Database 직접 확인

## 🔍 시나리오 6: 트렌드 수집 및 활용 (15분)

### 목표
YouTube/TikTok 트렌드를 수집하고 스크립트 생성에 활용합니다.

### 전제 조건
- Dashboard 로그인 상태
- Celery Worker 실행 중

### 단계별 진행

#### 6.1 트렌드 수집
1. Dashboard의 **Trends** 섹션
2. Region 입력: `KR` (또는 원하는 지역)
3. "트렌드 수집" 버튼 클릭
4. 처리 대기 (Celery 백그라운드 작업, 30초-1분)
5. 페이지 새로고침
6. **확인 사항**:
   - ✅ Trends 목록에 새 키워드 표시
   - ✅ 각 트렌드: keyword, platform, region, collected_at

#### 6.2 트렌드 필터링
1. Trends 목록 확인
2. Platform별 필터:

## 🎯 시나리오 7: Agent 모니터링 및 관리 (10분)

### 목표
등록된 Agent들을 모니터링하고 관리합니다.

### 전제 조건
- 최소 1개 이상의 Agent 등록됨
- Dashboard 로그인 상태

### 단계별 진행

#### 7.1 Agent 목록 확인
1. Dashboard의 **Agents** 섹션
2. 등록된 Agent 목록 확인
3. **확인 사항**:
   - ✅ Agent 이름, 상태, 마지막 heartbeat 표시
   - ✅ Online/Offline 상태 구분 (색상)

#### 7.2 Agent 통계 확인
1. Agents 섹션 상단의 통계 확인
2. **확인 사항**:
   - ✅ Total: 전체 Agent 수
   - ✅ Online: 현재 온라인 Agent 수
   - ✅ Offline: 오프라인 Agent 수

#### 7.3 Agent 상태 변화 테스트
1. Android 앱 종료
2. 약 1분 대기 (heartbeat 타임아웃)
3. Dashboard 새로고침
4. **확인 사항**:
   - ✅ Agent 상태가 "offline"으로 변경
   - ✅ 색상 변경 (녹색 → 회색)

#### 7.4 Agent 재시작
1. Android 앱 다시 실행
2. 자동으로 heartbeat 전송
3. Dashboard 새로고침
4. **확인 사항**:
   - ✅ Agent 상태가 "online"으로 복구
   - ✅ Last Heartbeat 업데이트

#### 7.5 Agent 디스크 정리 (선택적)
1. Agent 항목의 "디스크 정리" 버튼 클릭 (있는 경우)
2. 확인 창: "Agent의 임시 파일을 정리하시겠습니까?" → **확인**
3. **확인 사항**:
   - ✅ "디스크 정리 기록 완료" 메시지
   - ✅ Agent의 disk_usage_percent 감소 (있는 경우)

### 예상 소요 시간
10분

### 성공 기준
- Agent 목록/통계 정상 확인
- Agent 상태 변화 감지
- 온라인/오프라인 전환 확인

### 문제 발생 시
- Agent가 계속 offline: Android 앱 재시작, Backend URL 확인
- Heartbeat 업데이트 안됨: 네트워크 연결 확인

## 📝 시나리오 8: 멀티 Job 동시 처리 (30분)

### 목표
여러 Job을 동시에 생성하고 Agent가 순차적으로 처리하는 과정을 확인합니다.

### 전제 조건
- Agent 등록 및 online 상태
- Dashboard 로그인 상태

### 단계별 진행

#### 8.1 여러 Job 생성
1. Dashboard의 **Jobs** 섹션
2. Job 5개 생성:
   - Job 1: 제목 `테스트 1`, 우선순위 `5`
   - Job 2: 제목 `테스트 2`, 우선순위 `7` (높음)
   - Job 3: 제목 `테스트 3`, 우선순위 `3` (낮음)
   - Job 4: 제목 `테스트 4`, 우선순위 `5`
   - Job 5: 제목 `테스트 5`, 우선순위 `5`
3. **확인 사항**:
   - ✅ 5개 Job 모두 `pending` 상태
   - ✅ Jobs 통계: Pending count = 5

#### 8.2 우선순위별 처리 확인
1. Agent가 자동으로 Job 처리 시작
2. 처리 순서 관찰:
   - 예상 순서: Job 2 (우선순위 7) → Job 1, 4, 5 (우선순위 5) → Job 3 (우선순위 3)
3. **확인 사항**:
   - ✅ 높은 우선순위 Job이 먼저 처리됨
   - ✅ Agent는 한 번에 1개 Job만 처리

#### 8.3 Job 취소 테스트
1. `pending` 상태인 Job 선택
2. "취소" 버튼 클릭
3. 확인 창: "이 Job을 취소하시겠습니까?" → **확인**
4. **확인 사항**:
   - ✅ 상태가 `cancelled`로 변경
   - ✅ Agent가 해당 Job을 처리하지 않음

#### 8.4 Job 재시도 테스트
1. `failed` 또는 `cancelled` 상태인 Job 선택
2. "재시도" 버튼 클릭
3. **확인 사항**:
   - ✅ 상태가 다시 `pending`으로 변경
   - ✅ Agent가 다시 처리 시작

#### 8.5 모든 Job 완료 대기
1. 모든 Job이 `completed` 또는 `cancelled` 상태가 될 때까지 대기
2. Jobs 통계 확인
3. **확인 사항**:
   - ✅ Completed count 증가
   - ✅ Pending count = 0
   - ✅ 처리 시간 통계 표시

### 예상 소요 시간
30분 (Job 처리 시간)

### 성공 기준
- 여러 Job 동시 생성 가능
- 우선순위별 처리 순서 확인
- Job 취소/재시도 정상 동작

### 문제 발생 시
- Agent가 Job 처리 안함: Agent online 상태 확인
- 우선순위 무시됨: Backend Job 할당 로직 확인
