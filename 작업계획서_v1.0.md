### 2.2 주요 컴포넌트 역할

#### A. Agent Admin (FastAPI 기반)
- **트렌드 분석 파이프라인**
  - YouTube/TikTok API 스크래핑
  - Gemini: 시장 분석 + 구조 분석
  - Claude: 바이럴 스크립트 생성
  - Vector DB(pgvector): 콘텐츠 중복 방지 (유사도 85% 이하)

- **디바이스 관리 (1,000대)**
  - WebSocket 기반 실시간 상태 모니터링 (Heartbeat)
  - 작업 큐 관리 (Redis Queue)
  - OTA 업데이트 관리
  - User-Agent 계층 구조 관리

- **스케줄링**
  - 시간/요일별 자동 생성
  - 우선순위 기반 큐잉
  - 실패 재시도 로직

#### B. Agent (Android APK)
- **Hybrid 구조**
  - React Native: UI + 간단한 로직
  - Native Kotlin: FFmpeg 렌더링 + 접근성 API
  - Vibe-coding 방식 채택

- **영상 생성 프로세스**
  1. 정상 제목 text 수령 (다국어 지원: ko, en, ja, zh)
  2. 영상제목/콘텐츠/음악 수령
  3. 영상 스크립트 수령 (원본 언어 + 번역 언어)
  4. 로컬 FFmpeg + 하드웨어 가속 렌더링
  5. Google Cloud TTS 음성 합성 (언어별 Neural2 Voice)
  6. 자막 타이밍 자동 동기화 (SRT, 다국어 자막 생성)
  7. 언어별 버전 생성 (선택적)

- **디스크 관리**
  - 디스크 사용률 실시간 모니터링
  - 80% 이상 시 자동 정리 트리거
  - 우선순위: 임시 파일 → 30일 이상 영상 원본 → 렌더링 캐시
  - 히스토리 관리 불필요 (업로드 후 즉시 정리 가능)
  - Admin에 디스크 상태 보고 (Heartbeat 포함)

- **자동 업로드**
  - Accessibility Service (접근성 API) 활용
  - 사람의 터치/스와이프 모사
  - 업로드 후 딥링크 추출 → Admin에 Push

- **IP 분산**
  - 개별 SIM 카드 (모바일 데이터)
  - ADID, Mac Address 마스킹
  - 디바이스별 업로드 주기 랜덤화

#### C. Admin 서버 내부 모듈 구조

**1. API Layer (FastAPI)**
- REST API: Agent 작업 수신/보고
- WebSocket: 실시간 양방향 통신
- Dashboard API: 관리자 대시보드용

**2. Service Layer**
- **TrendService**: YouTube/TikTok API 스크래핑, Gemini 분석
- **JobService**: 작업 큐 관리, Agent 할당
- **StatsService**: 플랫폼 영상 성과 데이터 수집/집계
- **AnalyticsService**: 영상 수익성 분석 및 AI 피드백 생성
- **UploadLimitService**: 플랫폼별 업로드 제한 체크 및 스케줄링
  - 일일/시간당 업로드 횟수 추적
  - 최소 업로드 간격 검증
  - 제한 초과 시 자동 재스케줄링
  - 계정 로테이션 로직
- **OTAService**: APK 및 리소스 업데이트 배포

### 2.3 언어 선택 → 플랫폼 배포 워크플로우 (전체 플랫폼 동시 배포 방식)

#### 📋 전략 개요
- **컨셉**: 선택한 언어별로 영상 제작 후, 모든 활성화된 플랫폼에 동시 업로드
- **장점**: 
  - 구현 간단, 빠른 배포 가능
  - 최대 커버리지 확보 (8개 플랫폼 전체)
  - 플랫폼별 성과 비교 데이터 확보
  - 관리 및 모니터링 용이
- **단점**: 
  - 지역별 플랫폼 최적화 없음 (향후 Phase 2에서 추가 가능)

#### 🔄 상세 워크플로우

**1단계: 플랫폼 인증 정보 등록** (사용자)
```
Admin UI → 플랫폼 인증 관리 페이지
→ YouTube, TikTok, Instagram, Facebook 등 OAuth2/API Key 등록
→ user_platform_credentials 테이블 저장
```

**2단계: Channel 자동 생성** (시스템)
- 인증 정보 등록 시점에 **자동 실행**
- 해당 사용자가 소유한 **모든 Agent**에 대해 Channel 생성
- 예시: 100대 Agent 보유 + YouTube 인증 등록 → 100개 YouTube Channel 자동 생성

```sql
-- 인증 정보 등록 시 트리거
INSERT INTO channels (agent_id, user_credential_id, platform, status)
SELECT 
    a.id, 
    <새로_등록된_credential_id>, 
    '<platform_name>',
    'active'
FROM agents a
WHERE a.user_id = <current_user_id>
  AND NOT EXISTS (
    SELECT 1 FROM channels c 
    WHERE c.agent_id = a.id 
      AND c.platform = '<platform_name>'
  );
```

```
→ 시스템이 target_languages: ['ko', 'en', 'vi'] 저장
```

**4단계: Job 할당** (시스템)
- 유휴 상태 Agent 자동 선택
- `jobs.agent_id` 할당, `status='assigned'`
- Agent의 `/api/v1/jobs/available` API 호출 시 반환

**5단계: 영상 렌더링** (Agent)
- **언어별로 영상 생성** (3개 언어 → 3개 영상 파일)
  ```
  /storage/job_123_ko.mp4 (한국어 TTS, 자막)
  /storage/job_123_en.mp4 (영어 TTS, 자막)
  /storage/job_123_vi.mp4 (베트남어 TTS, 자막)
  ```
- Google Cloud TTS: 언어별 Neural2 Voice 사용
- FFmpeg 렌더링: 하드웨어 가속 (MediaCodec)

**6단계: 플랫폼 업로드** (Agent)
- **해당 Agent의 모든 활성화된 Channel에 업로드**
- 각 언어 × 각 플랫폼 조합으로 업로드 실행
- 예시: Agent #1이 YouTube, TikTok, Instagram Channel 보유
  ```
  Language: ko → Upload to [YouTube, TikTok, Instagram] (3건)
  Language: en → Upload to [YouTube, TikTok, Instagram] (3건)
  Language: vi → Upload to [YouTube, TikTok, Instagram] (3건)
  총 9개 영상 업로드
  ```

**7단계: 업로드 제한 체크** (시스템)
- 플랫폼별 일일/시간당 제한 실시간 추적 (Redis + PostgreSQL)
- 제한 초과 시:
  - 큐에 대기 (`upload_tracking.status='rate_limited'`)
  - 다음 가능 시점에 자동 재시도
  - Admin UI에 실시간 알림

```python
# 업로드 전 제한 체크 로직 (Pseudo Code)
def check_upload_limit(channel_id, platform):
    limit = get_platform_limit(platform)  # DB or Cache
    
    # 일일 제한 체크
    daily_count = redis.get(f"upload:daily:{channel_id}:{today}")
    if daily_count >= limit.daily_upload_limit:
        return {"allowed": False, "retry_at": tomorrow}
    
    # 시간당 제한 체크
    hourly_count = redis.get(f"upload:hourly:{channel_id}:{current_hour}")
    if hourly_count >= limit.hourly_upload_limit:
        return {"allowed": False, "retry_at": next_hour}
    
    # 최소 간격 체크
    last_upload = redis.get(f"upload:last:{channel_id}")
    if time_since(last_upload) < limit.min_upload_interval_minutes:
        return {"allowed": False, "retry_at": last_upload + interval}
    
    return {"allowed": True}
```

- 계정 밴 감지 시 해당 Agent 즉시 격리

```sql
CREATE EXTENSION IF NOT EXISTS vector;

-- Users 테이블
CREATE TABLE users (

    device_id VARCHAR(100) UNIQUE NOT NULL, -- MAC Address 해시
    api_key VARCHAR(128) UNIQUE NOT NULL,

    sim_carrier VARCHAR(50),
    android_version VARCHAR(20),
    apk_version VARCHAR(20),
    disk_usage_percent INT DEFAULT 0, -- 디스크 사용률 (0-100)

    platform_name VARCHAR(50) NOT NULL,

    auth_type VARCHAR(20) NOT NULL, -- 'oauth2', 'api_key', 'credentials'
    required_fields JSONB, -- 필수 인증 필드 정의 { "fields": ["client_id", "client_secret", "api_key"] }
    api_endpoint TEXT,
    documentation_url TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- User Platform Credentials 테이블 (User별 플랫폼 인증 정보)
CREATE TABLE user_platform_credentials (

    platform_id INT REFERENCES platforms(id) ON DELETE CASCADE,
    credential_name VARCHAR(100), -- 사용자 지정 이름 (예: "내 YouTube 계정 1")
    credentials JSONB NOT NULL, -- 암호화된 인증 정보 { "client_id": "...", "client_secret": "...", "api_key": "..." }
    is_default BOOLEAN DEFAULT FALSE, -- 기본 계정 여부
    status VARCHAR(20) DEFAULT 'active', -- 'active', 'expired', 'invalid'
    last_validated TIMESTAMP, -- 마지막 검증 시간
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(user_id, platform_id, credential_name)
);

-- Channels 테이블 (배포 채널)
CREATE TABLE channels (

);

    source_language VARCHAR(10) DEFAULT 'ko', -- 기본 제작 언어 (ko, en, ja, zh 등)

    error_message TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    assigned_at TIMESTAMP,
```

### 5.2 Redis 데이터 구조

```
# Job Queue (우선순위 큐)
ZADD job_queue <priority_score> <job_id>

# Agent Heartbeat (TTL 60초)
SETEX agent:heartbeat:<agent_id> 60 <timestamp>

# Available Agents (Set)
SADD agents:idle <agent_id>
SREM agents:idle <agent_id>

# API Rate Limiting (Sliding Window)
INCR ratelimit:<agent_id>:<minute>
EXPIRE ratelimit:<agent_id>:<minute> 60

# 플랫폼별 업로드 제한 추적 (일일)
HINCRBY upload:daily:<channel_id>:<YYYYMMDD> count 1
HINCRBY upload:daily:<channel_id>:<YYYYMMDD> size_mb <file_size>
EXPIRE upload:daily:<channel_id>:<YYYYMMDD> 86400

# 플랫폼별 업로드 제한 추적 (시간당)
INCR upload:hourly:<channel_id>:<YYYYMMDDHH>
EXPIRE upload:hourly:<channel_id>:<YYYYMMDDHH> 3600

# 마지막 업로드 시간 (최소 간격 체크용)
SETEX upload:last:<channel_id> 3600 <timestamp>

# Rate Limited 채널 (임시 차단)
SETEX upload:blocked:<channel_id> <seconds_until_reset> 1

# JWT Blacklist
SETEX jwt:blacklist:<token_id> <expiry_seconds> 1

# TTS Cache (24시간)
SETEX tts:cache:<text_hash> 86400 <audio_url>
```

     4. 시스템이 유휴 Agent에 Job 할당
     5. Agent가 언어별로 영상 렌더링 (N개 언어 → N개 파일)
     6. Agent가 각 언어 × 모든 활성화된 Channel에 업로드
     7. 시스템이 플랫폼별 업로드 제한 실시간 체크
     8. Agent가 업로드 결과 Admin에 보고
     9. Background Task가 플랫폼 API에서 성과 수집
     10. 시스템이 언어별/플랫폼별 성과 분석

3. ✅ **Channel 자동 생성 로직 명확화**
   - 인증 정보 등록 시점에 자동 실행
   - SQL 쿼리 예시 제공
   - 중복 생성 방지 로직

4. ✅ **업로드 제한 체크 로직 상세화**
   - Python Pseudo Code 제공
   - 일일/시간당 제한, 최소 간격 체크
   - Redis 기반 실시간 추적
   - Rate Limited 시 재시도 메커니즘

5. ✅ **API 응답 예시 추가**
   - Job 생성 요청 JSON 예시
   - Job 결과 보고 응답 JSON 예시 (언어별, 플랫폼별 업로드 결과 포함)

6. ✅ **Admin UI 언어 설정 페이지 명세**
   - 언어 체크박스 UI 상세 디자인
   - 예상 업로드 개수 자동 계산 표시
   - 활성화된 플랫폼 목록 표시

7. ✅ **성과 분석 SQL 쿼리 추가**
   - 언어별 성과 집계
   - 플랫폼별 성과 집계
   - 언어 × 플랫폼 매트릭스 분석 (최적 조합 발견)

8. ✅ **Phase 2 확장 로드맵**
   - 언어별 플랫폼 우선순위 설정
   - 지역별 계정 관리 (예: TikTok Vietnam vs Global)
   - 플랫폼별 콘텐츠 최적화
   - A/B 테스트 기능

**의사결정:**
- ✅ 전체 플랫폼 동시 배포 방식 채택 (Phase 1)
- ✅ 지역 최적화는 Phase 2로 연기 (데이터 기반 의사결정)

**기술적 세부 사항:**
- Channel 자동 생성: 인증 정보 등록 시 트리거
- 업로드 대상: `channels.status='active'` AND `user_platform_credentials.status='active'`
- 결과 추적: `videos` 테이블에 언어별×플랫폼별 개별 레코드 저장
- 성과 분석: 6시간마다 Background Task로 플랫폼 API 호출

**비즈니스 가치:**
- ✅ 사용자는 언어만 선택하면 모든 플랫폼에 자동 배포
- ✅ UX 단순화: 복잡한 플랫폼 선택 불필요

- ✅ 빠른 시장 검증: 어떤 언어×플랫폼 조합이 효과적인지 빠르게 파악
